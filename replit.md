# LogVision

Apache access log analytics dashboard — upload access.log files and explore historical usage patterns by application, time, and page.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/logvision run dev` — run the frontend (port varies, check workflow)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5 + multer (file uploads)
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Frontend: React + Vite + Tailwind + Recharts + Zustand

## Where things live

- `lib/api-spec/openapi.yaml` — source of truth for all API contracts
- `lib/db/src/schema/` — DB schema (logSessions, logEntries tables)
- `artifacts/api-server/src/routes/` — Express route handlers (logs, apps, stats)
- `artifacts/api-server/src/lib/logParser.ts` — Apache combined log format parser
- `artifacts/logvision/src/pages/` — Dashboard, AppDetail, Upload pages
- `artifacts/logvision/src/components/layout.tsx` — sidebar + session selector

## Architecture decisions

- Apache Combined Log Format parsed server-side; URL query strings stripped before storage for cleaner grouping
- First URL path segment used as "app name" (e.g. `/api/...` → `api`, `/shop/...` → `shop`)
- All filtering (by sessionId, hour range, app) handled via query params; no client-side aggregation
- Log entries stored in Postgres with indexed `hour`, `dayOfWeek`, `appName` columns for fast heatmap/timeline queries
- Orval zod output configured without TypeScript schemas (`schemas` option removed) to avoid barrel re-export collisions

## Product

- Upload Apache access.log files (up to 200MB, multipart)
- Switch between sessions with a dropdown in the sidebar
- Dashboard: global overview stats, timeline chart, status code pie
- App detail: day×hour heatmap, hourly bar chart, top pages table with hour range slider filter
- Applications automatically detected from URL path segments

## Gotchas

- After changing the OpenAPI spec, always run `pnpm --filter @workspace/api-spec run codegen` (the script post-processes the generated barrel to remove duplicate exports)
- `lib/api-zod/src/index.ts` is regenerated by codegen — do not hand-edit it
- DB schema push required after adding new tables: `pnpm --filter @workspace/db run push`
- File upload uses native `fetch`/XHR with FormData (not the generated Orval mutation hook) because Orval doesn't handle binary uploads
